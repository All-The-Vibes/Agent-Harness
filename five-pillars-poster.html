THE FIVE PILLARS OF A PRODUCTION HARNESS
01
CONTEXT
ASSEMBLY
Every model invocation receives a token payload maximizing comprehension per token.
  • Static: role + safety + format
  • Project: AGENTS.md, conventions
  • Dynamic: file tree, git status
  • Tool defs: MCP manifests
  • User msg: appended last (+30%)

02
TOOL
INTEGRITY
Every tool call is schema-validated before execution with actionable error messages.
  • Schema validation before effects
  • Poka-yoke parameter design
  • Output format optimization
  • Actionable error messages
  • ACI: rigor of user-facing API
FAILURE SIGNATURE
TOOL MISUSE
Bad tools, not bad model
03
LOOP
DISCIPLINE
Explicit signals — not heuristics — govern continue, retry, or halt.
  • ToolResult: success × continue
  • complete_task: explicit stop
  • Iteration ceiling (25 turns)
  • Token budget (80% trigger)
  • Idle detection + safety halt
FAILURE SIGNATURE
INFINITE LOOP
Nobody told it how to stop
04
POLICY
ENFORCEMENT
Every tool call passes through a permission gate before execution.
  • Read-only: ALLOW
  • Scoped write: ALLOW
  • Modifying source: CONFIRM
  • Destructive/external: CONFIRM
  • Credential access: DENY
FAILURE SIGNATURE
DESTRUCTIVE ACTIONS
Nobody built the gate
05
CONTEXT
LIFECYCLE
Context window actively managed as a finite, depletable resource.
  • HOT: prompt + recent turns
  • WARM: summarized older context
  • COLD: persisted to disk
  • Compaction at 80% utilization
  • Sub-agent delegation
FAILURE SIGNATURE
CONTEXT EXHAUSTION
Signal drowned by noise
